# nas-ql-monitor 扩展设计

日期：2026-05-22

## 目标
为 `nas-ql-monitor` 增加更适合青龙/NAS 长期运行的三类监控能力，并把 README 改成面向青龙面板的部署文档。

## 范围
1. 新增 `check_cert.sh`
   - 检查一个或多个 HTTPS 域名证书剩余天数
   - 支持阈值告警，默认 30 天
2. 新增 `check_backup.sh`
   - 检查一个或多个备份文件/目录是否存在
   - 检查最近修改时间是否超阈值
   - 可选最小文件大小校验
3. 新增 `daily_report.sh`
   - 汇总服务、容器、磁盘、证书、备份状态
   - 生成一条日报并通过现有通知模块发送
4. README 改为青龙专用说明
   - 拉库、任务命令、cron、环境变量、常见问题

## 设计原则
- 继续使用纯 shell，避免额外依赖
- 继续复用 `notify.sh`
- 默认以环境变量驱动，便于青龙任务直接覆盖
- 异常监控脚本默认“异常才通知”；日报脚本默认主动发送

## 输入约定
- `CERT_HOSTS`：空格分隔域名列表
- `CERT_WARN_DAYS`：证书告警天数阈值
- `BACKUP_TARGETS`：空格分隔检查项，格式 `路径|最大间隔小时|最小字节数`
- `REPORT_*`：日报汇总时所需目标列表

## 错误处理
- 单项失败不应中断整批检查；继续收集结果
- 无法解析目标时记为异常项
- 监控脚本退出码：有异常返回 1，无异常返回 0
- 日报脚本无论是否有异常都返回 0，避免被青龙误判失败
